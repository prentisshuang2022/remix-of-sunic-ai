## 3. 安全要求

### 3.1 认证与授权

| 项目 | 要求 |
|------|------|
| 认证方式 | JWT (JSON Web Token) |
| Access Token 有效期 | 2 小时 |
| Refresh Token 有效期 | 7 天 |
| Token 存储 | Access Token: 内存/sessionStorage；Refresh Token: httpOnly Cookie |
| 密码策略 | ≥ 8 位，含大写字母 + 小写字母 + 数字，建议含特殊字符 |
| 密码存储 | bcrypt (cost factor ≥ 12) |
| 暴力破解防护 | 连续失败 5 次锁定 30 分钟 |
| 登录异地告警 | IP 地理位置变化时推送告警（钉钉通知）[建议值，待确认] |
| 多端登录 | 允许同时登录，但同类设备最多 1 个活跃会话 [建议值，待确认] |

### 3.2 传输安全

| 项目 | 要求 |
|------|------|
| 全站 HTTPS | TLS 1.2+ |
| HSTS | `max-age=31536000; includeSubDomains` |
| CORS | 白名单限制，生产环境禁止 `*` |
| CSP | 按需配置 Content-Security-Policy |

### 3.3 数据安全

| 数据类型 | 存储方式 | 展示方式 | 日志方式 |
|----------|----------|----------|----------|
| 身份证号 | AES-256-GCM 加密（bytea） | 脱敏：`4301********1234` | 脱敏 |
| 银行卡号 | AES-256-GCM 加密（bytea） | 脱敏：`**** **** **** 1234` | 脱敏 |
| 手机号 | 明文存储 | 脱敏：`138****8888`（employee角色查他人） | 脱敏 |
| 邮箱 | 明文存储 | 脱敏：`abc****@example.com` | 脱敏 |
| 密码 | bcrypt hash | 永不展示 | 不记录 |
| 简历文件 | 对象存储 + 签名 URL | 签名 URL（1 小时有效） | 仅记录访问行为 |
| 员工附件 | 对象存储 + 签名 URL | 签名 URL（1 小时有效） | 仅记录访问行为 |

### 3.4 加密密钥管理

| 项目 | 建议方案 |
|------|----------|
| 加密密钥存储 | 环境变量 / 密钥管理服务（如 AWS KMS / Vault） |
| 密钥轮换 | 支持密钥版本化，旧密钥用于解密，新密钥用于加密 |
| 密钥备份 | 独立于数据库的安全备份 |

### 3.5 审计日志

必须记录的敏感操作：

| 操作类别 | 具体操作 |
|----------|----------|
| 员工信息 | 创建/修改/离职/调岗 |
| 敏感数据访问 | 查看完整身份证号/银行卡号 |
| 权限变更 | 角色分配/权限修改 |
| 招聘 | Offer 发送/候选人状态变更 |
| 绩效 | 评分提交/等级确定/申诉处理 |
| AI 操作 | 涉及敏感数据的 AI 查询/简历筛选结果 |
| 系统 | 登录/登出/密码修改/配置变更 |
| 数据导出 | 任何批量数据导出 |

审计日志保留 **2 年**，不可修改、不可删除。

## 4. 脱敏规则

| 数据类型 | 脱敏格式 | 脱敏算法 |
|----------|----------|----------|
| 身份证号 (18位) | `4301********1234` | 保留前4位+后4位，中间用 `*` 填充 |
| 手机号 (11位) | `138****8888` | 保留前3位+后4位 |
| 银行卡号 | `**** **** **** 1234` | 只保留后4位 |
| 邮箱 | `abc****@example.com` | 保留前3位+@后全部 |
| 姓名 | 不脱敏 | 非敏感字段 |

### 脱敏触发条件

| 场景 | 规则 |
|------|------|
| employee 查看他人 | 手机号、邮箱脱敏 |
| employee 查看自己 | 身份证脱敏展示，点击"查看完整"需二次验证 |
| manager 查看下属 | 手机号、邮箱不脱敏；身份证、银行卡不可见 |
| hr/admin | 全部字段完整展示 |
| API 响应 | 根据请求者角色自动脱敏 |
| 审计日志 | 所有敏感字段一律脱敏 |
| AI 上下文 | 传入 AI 前强制脱敏 |

## 5. 合规要求

### 5.1 数据保留与清理

| 数据类型 | 保留期限 | 清理方式 | 法规依据 |
|----------|----------|----------|----------|
| 在职员工数据 | 持续保留 | — | — |
| 离职员工数据 | 离职后 2 年 | 软删除 → 到期硬删除 | 《个人信息保护法》|
| 候选人数据 | 创建后 2 年 | 到期脱敏/删除 | 《个人信息保护法》|
| AI 对话记录 | 90 天活跃 + 归档 | 90 天后归档，归档后仅只读 | 内部审计需求 |
| 审计日志 | 2 年 | 到期可归档至冷存储 | 内部合规 |
| 面试反馈 | 同候选人 | 随候选人清理 | — |

### 5.2 个人信息保护

| 要求 | 说明 |
|------|------|
| 数据最小化 | 只采集业务必要的个人信息 |
| 知情同意 | 候选人上传简历视为同意处理（需隐私声明） |
| 数据可携带权 | 员工可申请导出自己的全部数据（JSON/Excel） |
| 删除权 | 候选人可要求删除个人信息（除法定保留义务） |
| 跨境传输 | AI API 如使用境外服务，需评估数据出境合规 [待确认] |

## 7. 可观测性

### 7.1 监控指标

| 类别 | 指标 | 告警阈值 |
|------|------|----------|
| API | 请求量 (QPS) | — |
| API | 响应时间 P95 | > 1s |
| API | 错误率 (5xx) | > 1% |
| API | 慢查询 | > 3s |
| AI | API 调用成功率 | < 95% |
| AI | 首字节延迟 P95 | > 5s |
| AI | 日 Token 消耗 | 超日预算 |
| AI | 月 Token 成本 | 超月预算 80% |
| DB | 连接池使用率 | > 80% |
| DB | 慢查询 | > 1s |
| 队列 | 积压消息数 | > 1000 |
| 队列 | 死信队列 | > 0 |
| 定时任务 | 执行状态 | 失败告警 |
| 定时任务 | 执行时长 | 超阈值告警 |
| 审批 | 超时审批数 | > 10 |
| 考勤 | 日异常数 | 环比 > 50% |

### 7.2 日志规范

| 级别 | 用途 | 示例 |
|------|------|------|
| ERROR | 系统错误、外部服务失败 | AI API 超时、数据库连接失败 |
| WARN | 业务异常、接近阈值 | Token 接近日限额、审批超时 |
| INFO | 关键业务操作 | 员工入职、Offer 发送、登录 |
| DEBUG | 开发调试（生产关闭） | 查询参数、中间计算 |

日志格式：结构化 JSON，包含 `timestamp`、`level`、`module`、`action`、`userId`、`traceId`、`duration`。

### 7.3 健康检查

| 端点 | 用途 | 检查项 |
|------|------|--------|
| `GET /health` | 存活探针 | 应用进程 |
| `GET /health/ready` | 就绪探针 | DB + Redis + 对象存储 |
| `GET /health/dependencies` | 依赖状态 | AI API + 钉钉 + MES/ERP |
